Write-up

Rationale for design decisions

We used three coordinated views because the question asks about change over time, regional comparison, and future scenario divergence. The main line chart emphasizes temporal trends. The bar chart makes it easier to compare regions at one selected year. The heatmap shows how strongly the three SSP futures separate from one another across regions.

The metric dropdown lets viewers switch between temperature anomaly, precipitation change, and a greenhouse gas concentration proxy. This supports dynamic querying because the same visual structure can answer related climate questions without overwhelming the viewer. The region filter reduces clutter in the line chart, while the year slider links all views together and helps users see how scenario differences become larger later in the century.

We chose line position on shared axes for trends because it is easier to read than area or pie encodings. Color is used for scenario because scenarios are categorical and important throughout the project. Tooltips provide details-on-demand, and zooming/panning supports closer inspection of specific time periods.

Data transformations

The original CMIP6 data is monthly gridded climate model output. We selected near-surface air temperature and precipitation, converted temperature from Kelvin to Celsius, converted precipitation from kg/m²/s to annualized mm/day values, averaged the data into broad world regions, and summarized monthly values into annual regional means. Temperature is shown as an anomaly from a 1995–2014 baseline, while precipitation is shown as percent change from the same baseline. Greenhouse gas concentration is represented as a scenario pathway proxy so the visualization can connect the climate responses to the emissions futures.

Development process

Our team first explored the CMIP6 dataset and considered maps, small multiples, and static line charts. We chose a coordinated interactive dashboard because it directly supports comparison across time, region, metric, and scenario. One person handled data processing, one person built the D3 interactions, and one person refined the write-up and design. Most time was spent reducing the large climate dataset into a small CSV that loads smoothly on GitHub Pages.
